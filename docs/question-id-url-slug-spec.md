# 問題ID・URLスラッグ規約（固定版）

GitHub Pages 向けの静的URL・サイトマップ・JSON-LD で共通利用する。**一度公開したスラッグは変更しない**（変更する場合は別途リダイレクト方針を定める）。

---

## 1. 用語

| 用語 | 意味 |
|------|------|
| **問題ID** | 1問を一意に指す **ASCII のドット区切り** 文字列。ログ・API・JSON で使う正規形。 |
| **URLパス** | リポジトリ上の相対パス。末尾は `index.html` を置く **ディレクトリURL** を想定。 |
| **スラッグ** | URL セグメントに使う `a-z` `0-9` `-` のみ（**小文字**）。 |

---

## 2. 科目（フィールド）スラッグ

CSV の「科目」列の値 → 次の固定マッピング（既存 `eisei2-master-data.js` / `csv_to_eisei2_master.py` の `FIELD_MAP` と同一）。

| CSV の値 | スラッグ | 意味 |
|-----------|----------|------|
| 関係法令 | `law` | |
| 労働衛生 | `rights` | |
| 労働生理 | `limit` | |

上記以外の科目が出たら **ビルドを失敗**させ、本表を更新してから再生成する。

---

## 3. 元号（過去問・開催年数）

CSV「開催年数」の先頭の和暦表記を次の **2〜3 文字 + 2桁年** に正規化する。

| パターン | スラッグ形式 | 例（CSV） | スラッグ |
|----------|--------------|-----------|----------|
| 令和 *N* 年 | `r` + 2桁 | 令和7年 | `r07` |
| 平成 *N* 年 | `h` + 2桁 | 平成31年 | `h31` |
| 昭和 *N* 年 | `s` + 2桁 | 昭和63年 | `s63` |

- *N* は十進整数。1桁年も **必ずゼロ埋め 2 桁**（令和1年 → `r01`）。
- 「令和7年（2025）」のような括弧付きは、先頭の `令和7年` 部分のみパースする。

**オリジナル**（開催年数が `オリジナル` の行）は §4 のとおり、問題ID に元号スラッグは **使わない**（ソースが `orig` で区別する）。

---

## 4. 開催月・プール（会期・オリジナルセット）

CSV「開催月」の文字列を、次の **会期スラッグ `session`** または **オリジナルプール `pool`** に変換する。

### 4.1 過去問（開催年数 ≠ オリジナル）

「開催月」に次の部分文字列が **含まれる**とき、**左から最初にマッチした**ルールを採用（`tools/csv_to_eisei2_master.py` の `month_bucket` と同順）。

| 開催月（部分一致） | session スラッグ |
|--------------------|------------------|
| カテゴリ別90 | `cat90` |
| 前期 | `zenki` |
| 後期 | `koki` |
| 10月 | `oct` |
| 4月 | `apr` |

いずれにもマッチしないが非空のとき: 前後空白を除いた全文を **NFKC 正規化**し、小文字化、`a-z0-9` 以外を `-` に置換、連続 `-` を1つにし、先頭末尾 `-` を削る。結果が空ならビルド失敗。

### 4.2 オリジナル（開催年数 = オリジナル）

同じく「開催月」を **プール `pool`** として上表にマッチさせる（過去問と同じ変換ルール）。

- 開催月が空・未対応で変換できない場合: プールは **`default`**（CSV に列を追加するまでの暫定。本番データでは空にしないことを推奨）。

---

## 5. 問番号

CSV「問番号」の整数を **10進**。URL では **ゼロ埋め 2 桁**（`01`〜`99`）。**100 問以上**になるプールでは **3 桁**（`001`）に繰り上げる（そのプール内で桁数を統一）。

問題ID文字列内では数値のみでもよいが、URL では常にゼロ埋め幅をプール内で統一する。

---

## 6. 問題ID（正規形・ドット区切り）

### 6.1 過去問

```
past.{era}.{session}.{field}.{問番号}
```

- `問番号`: ゼロ埋めなしの整数文字列でもよいが、**推奨はゼロ埋め2桁**（例 `03`）で統一。

**例:** 令和7年・後期・関係法令・問番号 3  
→ `past.r07.koki.law.03`

### 6.2 オリジナル

```
orig.{pool}.{field}.{問番号}
```

**例:** 開催月が「カテゴリ別90」・労働衛生・問番号 2  
→ `orig.cat90.rights.02`

---

## 7. URL パス（GitHub Pages・静的ファイル）

サイトルートからの相対パス。**末尾スラッシュ付きURL** に対応するため、各問は **ディレクトリ + `index.html`**。

### 7.1 過去問

```
q/past/{era}/{session}/{field}/{qwidth}/index.html
```

**例:**  
`q/past/r07/koki/law/03/index.html`  
公開URL例: `https://<user>.github.io/<repo>/q/past/r07/koki/law/03/`

### 7.2 オリジナル

```
q/orig/{pool}/{field}/{qwidth}/index.html
```

**例:**  
`q/orig/cat90/law/01/index.html`

`{qwidth}` は §5 のゼロ埋め問番号。

**ビルド:** `python3 tools/build_question_pages.py`（`--base-url` / `--site-prefix` は README 参照）

---

## 8. 正規URL（canonical）・OGP

- **canonical** は本番ドメイン確定後、`https://<本番ドメイン>/q/.../` 形式（末尾 `/` 統一）。
- **同一問の別表現**（クエリ付き・index.html 明示など）がある場合は、上記ディレクトリURLを canonical の正とする。

---

## 9. 既存の数値 `id` との関係

現行ビルドが付与する **数値 `id`**（`year * 1000 + ...` 等）は、アプリ内部・既存 localStorage 互換用として存続してよい。

- 新規実装では **問題ID（§6）を主キー**とし、数値 `id` は補助。
- 静的ページ生成時は **問題ID → パス（§7）** を一意に決める。
